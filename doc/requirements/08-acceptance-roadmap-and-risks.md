# 08. 验收、迭代与风险

## 1. V1 验收标准

### 1.1 场景生成验收

1. 每次能生成 100 个候选主题。
2. 系统能随机选择 1 个主题。
3. 最近 10 个主题能正确保存。
4. 生成新主题时会把最近 10 个主题传入 AI。
5. 生成的场景对话必须是英汉双语。
6. 场景 JSON 必须通过 Schema 校验。
7. 场景难度符合用户 CEFR 等级和 `difficultyScore`。

### 1.2 练习功能验收

1. 每个场景能生成三阶段练习。
2. 阶段一支持每句隐藏一个空。
3. 阶段二支持每句隐藏一半空。
4. 阶段三支持整句翻译。
5. 用户提交答案后能立即获得批改。
6. 批改结果包含得分、正确答案、错误原因和知识点标签。
7. 用户可以继续追问某道题。
8. 刷新页面后能恢复当前练习进度。

### 1.3 难度调整验收

1. 用户有 CEFR 等级。
2. 用户有内部 `difficultyScore`。
3. 完成场景后按分数调整难度。
4. 单次调整不超过 20 分。
5. 系统能根据最近成绩给出升降级建议。
6. 最近 5 个场景平均分控制跨等级调整。

### 1.4 复盘功能验收

1. 用户能查看历史场景。
2. 用户能查看每道题的历史答案。
3. 用户能查看错题。
4. 用户能重新练习错题。
5. 系统能总结最近薄弱点。
6. 系统能根据薄弱点推荐新练习方向。

### 1.5 技术验收

1. 前端通过 `npm run test:web`。
2. 后端通过 `npm run test:server`。
3. 冒烟测试覆盖登录、onboarding、生成场景、提交答案、完成场景。
4. AI JSON 输出有 schema 校验和失败重试。
5. 数据库迁移可从空库完整执行。
6. 核心接口有输入校验和错误响应。

### 1.6 UI 验收

1. Dashboard 第一屏直接展示学习状态和开始练习入口。
2. 练习页桌面端三栏布局可用，移动端能切换场景信息和辅助解析。
3. 主色、背景、边框、圆角符合 `07-ui-design-system.md`。
4. 所有工具按钮使用图标或图标加短文字，并有可访问标签。
5. 文本在移动端和桌面端不溢出、不遮挡。

## 3. 风险点与解决方案

### 3.1 AI 输出不稳定

风险：

AI 可能输出格式错误、字段缺失、难度不一致。

解决方案：

1. 使用严格 JSON Schema 或 Zod schema。
2. 输出失败自动重试。
3. 后端做字段校验。
4. 难度约束写入 Prompt。
5. 对话内容生成后进行二次质量检查。

### 3.2 难度波动过大

风险：

AI 生成内容有时过难，有时过简单。

解决方案：

1. 使用 CEFR + `difficultyScore`。
2. 使用明确句长、词汇、语法约束。
3. 单次难度调整限制在 ±20。
4. 最近 5 次平均成绩决定是否跨等级。
5. 保留人工修改难度入口。

### 3.3 场景重复

风险：

AI 可能反复生成类似场景。

V1 解决方案：

1. 保存最近 10 个场景主题。
2. 传给 AI 作为排除列表。
3. 禁止生成完全相同或明显相关主题。
4. 系统层面过滤完全相同标题。

V1 不做：

1. 向量相似度判断。
2. 语义相似度检测。
3. 自动聚类去重。

### 3.4 批改过严或过松

风险：

汉译英存在多种正确表达，AI 可能误判。

解决方案：

1. 阶段一、二尽量使用确定答案。
2. 阶段三采用多维度评分。
3. 允许自然表达。
4. 保存参考答案和可接受答案。
5. 批改 Prompt 明确“不能只做字符串匹配”。

### 3.5 用户学习挫败感

风险：

整句翻译太难，用户容易放弃。

解决方案：

1. 先隐藏一个空。
2. 再隐藏一半空。
3. 最后整句翻译。
4. 允许查看提示。
5. 错误解析要简洁、鼓励、可执行。
6. 连续低分自动降难度。

### 3.6 AI 成本与延迟

风险：

每轮生成 100 个主题、完整对话、练习题和批改，可能造成成本高、响应慢。

解决方案：

1. 主题生成和场景生成拆分，缓存候选主题。
2. 批改优先只处理当前题，不整场重算。
3. 对初始短测、复盘总结等非实时任务可以异步生成。
4. 增加 provider 超时和失败切换。
5. 记录任务耗时，用数据决定是否优化。

### 3.7 结构化数据膨胀

风险：

场景 JSON、答题记录、聊天线程长期增长，SQLite 文件变大。

解决方案：

1. 场景 JSON 作为快照保留，但列表页只查询索引字段。
2. 为 `userId`、`sceneId`、`createdAt`、`errorTags` 建索引。
3. 提供用户删除历史数据能力。
4. 后期可增加归档表或迁移到 PostgreSQL。

## 4. V1 里程碑建议

| 里程碑 | 目标 | 产出 |
|---|---|---|
| M1 工程骨架 | 跑通 Vue + Express + SQLite + Auth | 登录、迁移、健康检查 |
| M2 用户初始化 | 场景类别、生成模式、难度、初始短测 | Onboarding 和定级 |
| M3 场景生成 | 100 主题、随机选择、场景 JSON | SceneService 和 schema |
| M4 练习闭环 | 三阶段练习、提交、批改 | PracticeSession 和 Grading |
| M5 复盘闭环 | 历史、错题、薄弱点、难度调整 | Review 和 Weakness |
| M6 UI 打磨 | Dashboard、练习页、复盘页响应式 | 符合设计系统 |
| M7 验收发布 | 测试、冒烟、构建、部署说明 | V1 可试用版本 |

## 5. 成功指标

V1 可用性指标：

1. 新用户能在 3 分钟内进入第一轮练习。
2. 用户完成一轮场景练习后能看到清晰总结。
3. 每道错题都有错误标签和中文解释。
4. 完成 3 个场景后，系统能生成初步薄弱点画像。
5. 场景生成失败率低于 5%。
6. 批改失败率低于 3%。
