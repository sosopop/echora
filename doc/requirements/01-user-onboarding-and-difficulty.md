# 01. 用户初始化与难度体系

## 1. 场景类别设计

### 1.1 概念边界

场景类别用于描述用户希望练习内容主要落在哪些真实沟通场景中。它不是“学习目标”，不直接表达用户想提升的能力，而是影响 AI 生成场景时的主题范围。

```text
场景类别：练什么内容
难度体系：练到什么水平
薄弱点：重点补什么问题
```

用户可以选择一个或多个场景类别，也可以不选择任何类别，让 AI 在当前难度下随机生成真实交流场景。用户还可以输入自定义场景描述，最终由 AI 统一处理为标准场景主题和结构化对话。

### 1.2 场景类别类型

| 场景类别 | 说明 |
|---|---|
| `daily_english` | 日常交流 |
| `travel_english` | 旅游英语 |
| `business_english` | 商务英语 |
| `interview_english` | 面试英语 |
| `campus_english` | 校园英语 |
| `shopping_english` | 购物消费 |
| `restaurant_english` | 餐厅点餐 |
| `medical_english` | 医疗就诊 |
| `phone_english` | 电话沟通 |
| `technical_english` | 技术交流 |
| `social_english` | 社交闲聊 |
| `life_abroad` | 海外生活 |
| `exam_expression` | 考试表达 |

### 1.3 场景类别对 AI 生成的影响

AI 生成场景时应综合以下条件：

```json
{
  "userLevel": "B1",
  "difficultyScore": 420,
  "generationMode": "category_constrained",
  "scenarioCategories": ["daily_english", "business_english"],
  "categoryMatchThreshold": 0.7,
  "recentSceneTopics": [
    "ordering coffee at a cafe",
    "asking for directions in a city"
  ],
  "weaknessTags": ["past_tense", "preposition", "word_order"]
}
```

生成结果应优先满足：

1. 符合场景类别约束，或在无约束模式下生成自然多样的真实场景。
2. 符合当前英语水平。
3. 避免最近 10 个场景主题。
4. 适当覆盖用户薄弱点。
5. 场景具有真实交流价值。

### 1.4 场景生成模式

| 模式 | 字段值 | 说明 |
|---|---|---|
| 按类别生成 | `category_constrained` | 用户选择一个或多个场景类别，AI 按类别生成候选主题 |
| 随机生成 | `open_random` | 用户不做场景约束，AI 根据难度、薄弱点和最近主题随机生成 |
| 自定义场景 | `custom_scenario` | 用户输入自然语言场景，AI 先规范化为主题，再生成对话和练习 |

### 1.5 类别阈值

场景类别可以使用匹配阈值控制约束强度。阈值越高，AI 生成主题越贴近所选类别；阈值越低，允许更发散的真实场景。

| 阈值 | 说明 | 示例 |
|---:|---|---|
| `1.0` | 强约束，候选主题必须明确属于所选类别 | 只选 `restaurant_english` 时，主题必须围绕餐厅沟通 |
| `0.7` | 默认约束，主题以所选类别为主，可包含相邻场景 | 餐厅点餐可延展到预订、投诉、结账 |
| `0.4` | 弱约束，优先但不强制贴合类别 | 日常交流可混入购物、交通、社交 |
| `0` 或未设置 | 不进行类别约束 | AI 在当前难度下随机生成真实交流场景 |

### 1.6 自定义场景

用户可以输入自定义场景，例如：

```text
我想练习在公司里向同事解释项目延期，并尽量表达得礼貌一点。
```

系统不直接把用户输入当作最终场景，而是交给 AI 处理为标准结构：

```json
{
  "generationMode": "custom_scenario",
  "customScenarioText": "我想练习在公司里向同事解释项目延期，并尽量表达得礼貌一点。",
  "normalizedTheme": {
    "titleEn": "Explaining a project delay to a coworker politely",
    "titleZh": "礼貌地向同事解释项目延期",
    "category": "business_english",
    "estimatedLevel": "B1"
  }
}
```

自定义场景仍必须遵守：

1. 当前 CEFR 难度约束。
2. 最近场景去重规则。
3. 内容安全和真实交流价值。
4. 场景 JSON Schema。

## 2. 难度体系设计

### 2.1 双层难度体系

#### 用户可见难度

| 用户显示等级 | 对应 CEFR |
|---|---|
| 入门 | A1 |
| 初级 | A2 |
| 中级 | B1 |
| 中高级 | B2 |
| 高级 | C1 |
| 精通 | C2 |

#### 系统内部难度分

系统内部维护 `difficultyScore`，范围为 0-1000。

| 分数范围 | CEFR |
|---|---|
| 0-149 | A1 |
| 150-299 | A2 |
| 300-499 | B1 |
| 500-699 | B2 |
| 700-849 | C1 |
| 850-1000 | C2 |

### 2.2 CEFR 生成约束

AI 生成内容时，必须参考确定性约束，不能只依赖“简单/中等/困难”这类模糊描述。

| 等级 | 句长 | 对话轮次 | 语法 | 词汇与表达 |
|---|---:|---:|---|---|
| A1 | 5-8 个英文词 | 4-6 轮 | be 动词、一般现在时、简单疑问句 | 高频生活词，中文含义简单直接，避免从句和复杂时态 |
| A2 | 6-10 个英文词 | 6-8 轮 | 一般现在时、一般过去时、一般将来时 | 常见生活和旅行表达，可包含 because、but、so |
| B1 | 8-14 个英文词 | 8-10 轮 | 现在完成时、情态动词、宾语从句 | 日常、职场、校园常用表达，可表达原因、建议、计划、偏好 |
| B2 | 12-18 个英文词 | 10-12 轮 | 条件句、定语从句、被动语态 | 商务、抽象话题、观点表达，支持解释、对比、协商、委婉表达 |
| C1 | 15-24 个英文词 | 12-16 轮 | 复杂从句、让步结构、强调结构 | 高级表达、抽象词汇、语域变化，支持逻辑论证和细腻态度 |
| C2 | 18-30 个英文词 | 12-18 轮 | 高级句式、复杂修辞、自然口语省略 | 接近母语者真实表达，支持复杂观点、幽默、反讽、隐喻 |

### 2.3 难度约束使用规则

1. 场景生成、对话生成、练习题生成都必须携带 `userLevel` 和 `difficultyScore`。
2. `difficultyScore` 用于在同一个 CEFR 等级内部微调难度。
3. AI 输出后，后端需要做结构校验和基础质量校验。
4. 如果 AI 输出明显不符合当前 CEFR 约束，应触发重试或降级修复。
5. 用户可以手动调整可见等级，但系统仍保留历史分数和调整记录。

## 3. 初始水平测试

### 3.1 初始流程

新用户第一次进入时：

1. 用户选择自认为的难度等级。
2. 系统生成 8-12 道短测题。
3. 题型仍采用 V1 的汉译英模式。
4. AI 根据结果判断用户实际等级。
5. 系统生成初始 `difficultyScore`。

### 3.2 初始测试题型

| 类型 | 数量 |
|---|---:|
| 每句隐藏一个空 | 3-4 题 |
| 每句隐藏一半空 | 3-4 题 |
| 整句翻译 | 2-4 题 |

### 3.3 初始定级规则

系统根据用户自选等级和短测表现共同决定初始等级：

```text
初始分数 = 用户自选等级基础分 + 测试修正分
```

#### 用户自选等级基础分

| 用户自选等级 | 基础分 |
|---|---:|
| 入门 A1 | 80 |
| 初级 A2 | 220 |
| 中级 B1 | 400 |
| 中高级 B2 | 600 |
| 高级 C1 | 780 |

#### 测试修正

| 测试得分 | 修正 |
|---|---:|
| ≥ 90 | +80 |
| 80-89 | +40 |
| 70-79 | +10 |
| 60-69 | -20 |
| < 60 | -60 |

### 3.4 定级限制

1. 初始测试最多上调一个 CEFR 等级。
2. 初始测试最多下调一个 CEFR 等级。
3. 用户可手动修改系统建议等级。
4. 如果用户跳过短测，则以自选等级基础分进入，并在前 3 个场景中加大动态调节权重。

## 4. 新用户引导页要求

### 4.1 页面步骤

| 步骤 | 页面内容 | 核心操作 |
|---|---|---|
| 账号入口 | 登录、注册、游客体验入口 | 完成认证或进入体验模式 |
| 场景类别 | 多选场景类别、选择不约束、输入自定义场景 | 保存 `scenarioCategories`、`generationMode`、`customScenarioText` |
| 自选等级 | 选择入门到精通 | 写入初始等级候选 |
| 初始短测 | 8-12 道汉译英题 | 生成初始 `difficultyScore` |
| 结果确认 | 展示等级、薄弱点、推荐练习 | 进入 Dashboard |

### 4.2 引导体验原则

1. 不展示复杂理论，使用短句解释“为什么要选择场景类别和难度”。
2. 用户完成短测后应获得具体、温和、可执行的反馈。
3. 允许跳过短测，但需要说明系统会在后续练习中自动校准。
4. 首次进入核心练习前，用户应清楚知道本轮练习的场景、阶段和目标。
