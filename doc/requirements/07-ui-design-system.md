# 07. UI 设计系统

## 1. 设计方向

Echora 是一个 AI 英语情景练习工具，不是营销型落地页。第一屏应直接服务学习工作流：继续练习、今日推荐、当前水平、薄弱点和最近场景。

UI 风格参考 [getdesign.md Claude Design System Analysis](https://getdesign.md/claude/design-md) 中的方向：暖陶土色强调、干净的编辑式布局、纸张感背景、温和边框和克制阴影。该参考页说明其风格适合 AI chat interfaces、warm editorial layouts 和 intellectual product pages，Echora 可借用这种“安静、可信、适合长时间阅读和练习”的视觉语言。

## 2. 视觉关键词

| 关键词 | Echora 落地方式 |
|---|---|
| 温暖 | 使用暖陶土色和纸张背景，降低学习压力 |
| 编辑感 | 信息按阅读节奏排布，避免装饰性大块视觉 |
| 工具感 | Dashboard 和练习页保持高可扫读性 |
| AI 感 | 用对话流、解析面板和结构化反馈表达智能能力 |
| 克制 | 不使用大面积渐变、浮夸阴影、装饰光斑 |

## 4. 字体与排版

### 4.1 字体栈

```css
:root {
  --font-serif: Georgia, "Times New Roman", "Noto Serif SC", "Songti SC", serif;
  --font-sans: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
    "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
  --font-mono: "SFMono-Regular", Consolas, "Liberation Mono", monospace;
}
```

使用原则：

1. 产品导航、按钮、表单、状态标签使用 sans。
2. 场景说明、复盘摘要、长解释可以少量使用 serif，形成编辑感。
3. 英文句子使用 sans，保持可读和易输入。
4. 代码、JSON、Prompt 示例使用 mono。

### 4.2 字号

| Token | 字号 / 行高 | 用途 |
|---|---|---|
| `--text-page-title` | 32px / 1.2 | Dashboard 或页面标题 |
| `--text-section-title` | 24px / 1.3 | 页面分区标题 |
| `--text-panel-title` | 18px / 1.4 | 卡片、面板标题 |
| `--text-body` | 16px / 1.6 | 常规 UI 和说明 |
| `--text-english` | 18px / 1.55 | 英文句子、用户输入回显 |
| `--text-chinese` | 16px / 1.6 | 中文题干和解析 |
| `--text-caption` | 14px / 1.43 | 元信息、提示 |
| `--text-label` | 12px / 1.6 | 标签、表头、小型控件 |

## 5. 间距、圆角与层级

### 5.1 间距

以 8px 为基础单位，常用值：

```css
--space-0-5: 4px;
--space-1: 8px;
--space-1-5: 12px;
--space-2: 16px;
--space-3: 24px;
--space-4: 32px;
--space-6: 48px;
```

### 5.2 圆角

| Token | 数值 | 用途 |
|---|---:|---|
| `--radius-sm` | 4px | 标签、小输入 |
| `--radius-md` | 6px | 按钮、分段控件 |
| `--radius-lg` | 8px | 卡片、面板、弹窗 |
| `--radius-xl` | 12px | 仅用于大型浮层，不作为默认 |

默认卡片圆角不超过 8px。

### 5.3 层级

| 层级 | 样式 | 用途 |
|---|---|---|
| Flat | 无边框、无阴影 | 页面背景和普通文本区 |
| Contained | `1px solid var(--color-border-cream)` | 标准面板 |
| Ring | `0 0 0 1px var(--color-ring-warm)` | 可交互卡片 hover |
| Whisper | `0 4px 24px rgba(20, 20, 19, 0.05)` | 重点浮层、辅助窗口 |
| Inset | `inset 0 0 0 1px rgba(20, 20, 19, 0.12)` | 按下态 |

## 6. 组件规范

### 6.1 按钮

| 类型 | 样式 | 用途 |
|---|---|---|
| Primary | 陶土色底、白字、6px 圆角 | 开始练习、提交答案 |
| Secondary | 暖沙色底、深色文字 | 取消、返回、次要操作 |
| Ghost | 透明底、hover 显示暖沙背景 | 面板工具按钮 |
| Danger | 错误红底或红色描边 | 删除历史、重置 |
| Icon | 正方形 36-40px，lucide 图标 | 解析、收藏、设置、关闭 |

规则：

1. 工具类操作优先使用 lucide 图标，并提供 tooltip。
2. 主要操作按钮文字保持短，不在按钮里放长解释。
3. loading 状态要锁定宽度，避免按钮跳动。

### 6.2 输入框

1. 默认高度 40px，长答案文本框最小高度 96px。
2. 聚焦态使用 `--color-focus-blue` ring，保证可访问性。
3. 错误态使用红色边框和简短错误说明。
4. 英文答题输入框应使用 18px 字号，方便用户检查拼写。
5. 答案输入支持 Enter 提交、Shift+Enter 换行。

### 6.3 标签

| 类型 | 示例 | 视觉 |
|---|---|---|
| 阶段 | `阶段 2` | 陶土色浅底 |
| 等级 | `B1` | 深色文字 + 暖边框 |
| 错误标签 | `preposition` | 错误红浅底 |
| 目标 | `商务英语` | 暖沙底 |
| 状态 | `批改中` | 中性色浅底 |

### 6.4 卡片与面板

1. 卡片用于重复项目、记录条目、弹窗、真正需要框定的工具区域。
2. 页面大分区不套大卡片，使用全宽背景带或无框布局。
3. 禁止卡片套卡片。
4. 面板内标题应使用 18px 或 16px，不用页面级大标题。

### 6.5 对话气泡

| 类型 | 样式 |
|---|---|
| AI 出题 | Ivory 表面，左侧细陶土色线 |
| 用户回答 | 白色表面，暖边框，右对齐或同列高亮 |
| 批改结果 | 根据正确/错误显示绿色或红色状态条 |
| 系统提示 | 暖沙浅底，小字号 |

## 7. 页面结构

### 7.1 首页 Dashboard

首页应展示：

1. 当前等级。
2. 当前 `difficultyScore`。
3. 最近 7 天练习次数。
4. 最近平均分。
5. 今日推荐练习。
6. 高频错误标签。
7. 最近场景列表。
8. 继续上次练习入口。

布局原则：

1. 顶部保留紧凑导航和用户状态。
2. 第一屏左侧是“继续练习 / 今日推荐”，右侧是学习状态摘要。
3. 趋势图和错题排名放在下方，避免抢核心操作。
4. 不做大 hero，不使用宣传文案占据首屏。

### 7.2 场景练习页

桌面布局：

```text
┌────────────────────────────────────────────────────────┐
│ 顶部：Echora / 当前等级 / 当前阶段 / 本场得分          │
├───────────────┬────────────────────────┬───────────────┤
│ 场景信息侧栏   │ 主聊天练习区             │ 辅助解析窗口   │
│ 角色 / 目标    │ AI 出题                 │ 句子解析       │
│ 阶段进度       │ 用户答题                 │ 语法追问       │
│ 重点表达       │ 即时批改                 │ 扩展表达       │
└───────────────┴────────────────────────┴───────────────┘
```

移动端布局：

1. 顶部展示场景标题、阶段和进度。
2. 主练习区优先显示。
3. 场景信息和辅助解析通过底部抽屉或 tabs 打开。
4. 提交按钮固定在输入区域附近，不遮挡内容。

### 7.3 复盘页

复盘页采用数据工具布局：

1. 顶部统计：平均分、练习次数、错题数、当前等级。
2. 中部趋势：得分趋势、难度变化、错误标签排名。
3. 下方列表：历史场景、错题、推荐重练。
4. 用户可按标签、场景、阶段筛选。

### 7.4 错题本页

错题本需要支持：

1. 按错误标签筛选。
2. 按阶段筛选。
3. 按场景筛选。
4. 原题重练。
5. 变体重练。
6. 打开辅助解析。

## 8. 交互状态

### 8.1 AI 生成中

1. 使用明确的进度状态，例如“正在生成候选场景”“正在构造对话”“正在生成练习题”。
2. 允许取消或重新生成，取消后不写入场景历史。
3. 生成时间超过预期时展示重试入口。

### 8.2 批改中

1. 提交后锁定当前答案，避免重复提交。
2. 批改失败时保留答案并允许重试。
3. 批改结果出现后，焦点移到结果区域，方便键盘用户阅读。

### 8.3 空状态

空状态应短而具体：

| 场景 | 文案方向 |
|---|---|
| 无历史场景 | “完成第一轮练习后，这里会出现你的场景记录。” |
| 无错题 | “最近没有错题，可以继续挑战新场景。” |
| 无薄弱点 | “练习数据还不够，完成 3 个场景后会生成分析。” |

## 9. 可访问性与响应式

1. 所有图标按钮必须有 `aria-label` 或 tooltip。
2. 颜色不能作为唯一状态提示，正确/错误需要配合文字和图标。
3. 键盘可完成答题、提交、下一题、打开解析。
4. 字体不随 viewport 等比缩放。
5. 桌面主练习区最小宽度应保证英文句子不被挤压。
6. 移动端避免三栏挤压，使用 tabs 或抽屉。

## 10. 禁用方向

1. 不使用大面积紫蓝渐变。
2. 不使用装饰性光斑、漂浮圆球、模糊背景。
3. 不做营销式 hero 占满首屏。
4. 不把所有内容都包进多层卡片。
5. 不让文本覆盖按钮、图表或输入区域。
6. 不在应用内用长文解释“这个功能怎么用”，通过界面状态和简洁提示引导。
