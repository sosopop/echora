Learning Flow

状态机驱动的 AI 学习流。

主学习流只有一条。AI 负责生成、解释和推荐；系统负责保存、提交、锁定、权限和状态转换。

MVP Action Matrix

每个状态的可落地主线动作

这里保留 MVP 必需分支；不清晰或容易扩大范围的动作不进入主线，只能作为辅助追问解释。

状态 允许动作 系统结果
onboarding 自然回答 / 跳过 / 登录或注册保存 补齐画像后进入 scene_selecting
scene_selecting 选择场景 / 输入自定义场景 / 换一批 锁定场景上下文，进入 practicing
practicing 提交答案 / 要提示 / 跳过 / 打开辅助追问 正式答案进入 grading；辅助追问不改主线
grading 等待批改 / 查看处理过程 / 打开辅助追问 写入批改与错误标签，进入 awaiting_next
awaiting_next 下一题 / 类似题 / 换场景 / 查看复盘 分别进入 practicing、scene_selecting 或 reviewing
reviewing 继续练习 / 重练薄弱点 / 换场景 / 查看历史 可创建新学习流或回到练习主线
archived 只读查看 / 引用摘要新建学习流 不可继续答题，只能创建新的 active 会话
Product Tree

主线稳定，分支开放生长

这里展示需求审计中产生的发散方向。只有 MVP 保留和 MVP 收敛会进入真实演示；候选、暂不做和已砍掉项只作为取舍记录。

MVP 保留

单一主学习流

场景、练习、批改、复盘和重练都在中间学习流完成，底部交互区是唯一主线输入入口。

MVP 收敛

账号入口 Widget

登录、注册、保存进度以 account-gate 出现，但真实鉴权和保存由系统执行，不交给 AI 自由处理。

MVP 保留

辅助追问不改主线

右侧支线只能解释来源上下文；想生成新题、换场景或进入复盘，必须回到主学习流确认。

后续候选

长期学习计划

可在后续版本基于薄弱点和历史目标生成周计划，但 V1 不依赖计划系统完成学习闭环。

暂不做

复杂账号表单

V1 原型不做真实表单校验、密码处理、第三方登录或家长账号绑定，只表达保存入口。

已砍掉

旧式多页面学习平台

不把错题本、设置页、复盘页恢复成主导航。它们的能力必须长在学习流和 Widget 里。

AI 建议 + 系统确定动作

AI 可以建议练什么、怎么解释、下一步去哪；系统负责提交答案、保存、登录、新建会话、锁定历史和权限判断。

1

2

系统校验当前状态

3

AI 生成内容或建议

辅助追问不改变主线

右侧追问只解释来源上下文。用户想换场景、出新题或复盘时，必须回到主学习流确认。

来自：这道题

解释 order / book / reserve 的区别，主线仍停留在当前题。

练习中历史锁定

当前题未完成时，旧会话可见但答案、参考答案和批改详情隐藏或只读，避免用户回看答案影响练习。

暂不可查看完整答案

完成当前题后恢复完整复盘。

学习菜单触发结构化动作

用户只看到普通菜单。点击后系统决定直接执行、展示确认，或交给 Echo 生成下一步内容。

登录 / 注册 / 保存入口

登录和注册都可以作为对话内小部件出现，但真实鉴权、保存和恢复仍由系统执行。

登录或注册后保存这次学习进度

账号动作在对话里发起，由系统完成鉴权与保存。

低置信度意图确认

当 Echo 不确定用户想做什么时，只展示自然选项，不展示分数、路由或内部判断。
