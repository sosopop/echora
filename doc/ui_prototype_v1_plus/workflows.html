Learning Flow

状态机驱动的 AI 学习流。

主学习流只有一条。AI 负责生成、解释和推荐；系统负责保存、提交、锁定、权限和状态转换。

Current State

onboarding

Echo 用自然对话收集用户画像，系统保存必要字段，并在完成后进入场景选择。

系统负责

保存画像、校验账号入口、创建当前会话。

AI 负责

询问关键信息、总结画像、推荐下一步。

account-gate intent-confirm

AI 建议 + 系统确定动作

AI 可以建议练什么、怎么解释、下一步去哪；系统负责提交答案、保存、登录、新建会话、锁定历史和权限判断。

辅助追问不改变主线

右侧追问只解释来源上下文。用户想换场景、出新题或复盘时，必须回到主学习流确认。

来自：这道题

解释 order / book / reserve 的区别，主线仍停留在当前题。

练习中历史锁定

当前题未完成时，旧会话可见但答案、参考答案和批改详情隐藏或只读，避免用户回看答案影响练习。

暂不可查看完整答案

完成当前题后恢复完整复盘。

学习菜单触发结构化动作

用户只看到普通菜单。点击后系统决定直接执行、展示确认，或交给 Echo 生成下一步内容。

登录 / 注册 / 保存入口

登录和注册都可以作为对话内小部件出现，但真实鉴权、保存和恢复仍由系统执行。

登录或注册后保存这次学习进度

账号动作在对话里发起，由系统完成鉴权与保存。

低置信度意图确认

当 Echo 不确定用户想做什么时，只展示自然选项，不展示分数、路由或内部判断。
