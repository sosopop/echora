# PRD 剩余缺口清单

> 日期: 2026-05-18
> 基线: 任务 048 完成后的代码与文档状态

## 说明

这份清单只记录截至当前仍未完全收口的项。它们大多属于工程加固、协议现代化或体验增强，不代表四阶段主学习主线没有完成。

## 剩余缺口

1. **SSE 传输现代化与多副本恢复**（PRD §2.8 / §3.4 / §5.1）
   - 现状: 目前仍以 `EventSource + ?token=` 为主,进程内 `streamBus` 负责热回放,047 已补历史快照兜底。
   - 缺口: 迁移到 `fetch + ReadableStream`,补标准 `Last-Event-ID` 支持,并把多副本场景下的 replay backend 升级为 Redis Streams 或等价共享流。
   - 价值: 降低对单进程内存的依赖,把断线恢复和部署弹性一起补齐。

2. **归档会话的模板化派生**（PRD §3.1 / §5.2）
   - 现状: 完成一轮后会 rollover 到新会话,旧会话保持只读归档。
   - 缺口: 定义并实现“从 archived 会话复制为新学习流模板/上下文”的正式入口。
   - 价值: 支持历史复用,但不破坏锁定与归档边界。

3. **生成取消信号的全链路传播**（PRD §3.5）
   - 现状: 前端停止按钮和 `/api/chat/streams/:streamId/abort` 已有,主线与支线的 provider 调用已开始接入 `AbortSignal`,但仍需继续核对所有长运行技能路径。
   - 缺口: 把取消信号贯穿到所有长运行 skill 与 provider 调用。
   - 价值: 停止生成更及时,减少无效 token 与“卡住”的感觉。

4. **Widget 样式目录拆分**（工程收尾项）
   - 现状: widget 相关样式还集中在现有公共样式与少量模块里。
   - 缺口: 是否拆出 `src/styles/widgets/` 子目录仍未最终定案。
   - 价值: 提高 widget 体系后续扩展时的样式可维护性。

## 结论

截至当前,PRD 的核心主干已经完成,剩余项主要集中在协议现代化、归档复用边界和少量工程收尾。后续推进可以优先从 **SSE 现代化**、**归档会话模板化派生** 和 **取消信号全链路传播** 这三项继续。
