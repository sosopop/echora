# Echora V1 MVP — 产品需求文档

> 版本：V1 MVP · 更新日期：2026-05-16

## 1. 概述

### 1.1 产品定位

Echora 是一个英语场景对话练习的 AI Agent 应用。AI 扮演英语教练，围绕真实生活场景，通过对话驱动学习全流程。用户感知到的始终是"和一位聪明的英语教练在真实场景中持续练习"，而非操作一个学习管理平台。

### 1.2 一句话描述

一个对话聊天界面承载所有英语学习功能。用户通过底部交互区和对话内容列表完成学习、练习、批改、复盘和历史回顾，桌面端可展开左侧历史会话和右侧辅助追问辅助管理上下文。

### 1.3 核心设计理念

- **对话即界面**：学习、练习、批改、复盘和历史回顾主要在一个对话界面完成。账号、隐私授权、数据删除等敏感操作可由系统级 Widget 承载，敏感字段不进入普通消息上下文
- **AI 会话流式学习**：学习过程连续、自然、顺畅。AI 根据当前状态提出下一步建议，系统状态机负责新建/归档会话、锁定历史答案、保存进度等确定性动作
- **小部件即功能**：Widget 是对话中的结构化交互单元。AI 可建议展示场景卡片、练习题、批改结果、账号提示等 Widget，系统负责校验 Widget schema、权限和可执行动作
- **辅助追问**：普通对话一旦中途插话就会被带偏方向，辅助追问解决了这个问题——用户可以在主学习流不变的前提下，随时对当前或之前的任意消息、题目、批改结果打开右侧支线深入追问，问完回到主线继续
- **Profile 驱动**：基于用户画像个性化生成场景和学习路径
- **场景沉浸式对话**：所有练习围绕真实生活场景（餐厅点餐、校园对话、旅行问路等）展开，用户在情境中学习表达而非孤立背单词
- **学习闭环驱动**：场景选择 → 练习目标 → 作答 → 批改归因 → 更新掌握度/薄弱点 → 复盘或重练 → 再测达标，形成可追踪闭环。每次练习和批改结果结构化记录，支撑精准复盘和智能重练
- **自适应难度**：AI 与规则共同根据实时表现调整题型、难度和提示强度，连续答对逐步提升挑战，连续答错降低门槛并补充引导，用户也可随时说“太难/太简单”

### 1.4 技术选型

- **前端**：React 18 + TypeScript + Vite + Zustand
- **后端**：Node.js + Express 5 + TypeScript
- **数据库**：SQLite + better-sqlite3
- **流式通信**：SSE (EventSource)
- **校验**：Zod
- **鉴权**：JWT + bcryptjs
- **AI**：Provider 抽象层，支持多模型切换

## 2. 基础功能

### 2.1 用户旅程

```
新用户注册/登录
  → AI 对话式收集画像（姓名、年龄、年级、英语水平）
  → AI 推荐 3-5 个个性化场景卡片
  → 用户选择场景
  → 练习（5-10 题/轮，题型自动匹配水平）
  → 批改（流式反馈 + 错误标签）
  → 可打开辅助追问深入理解
  → 更新掌握度并复盘总结
  → 循环或换场景
```

### 2.2 Skill 体系（8 个 Skill）

每个功能封装为独立 Skill，注册到运行时 `skillRegistry`，AI 可在系统允许范围内调用。Skill 可关联 Widget 和输入模式，通过对话流中的小部件呈现给用户。新增功能 = 新增 Skill + 注册，无需修改路由或核心框架。

- **`onboarding`**：对话式收集用户画像，映射 CEFR 等级。首次登录自动触发。
- **`scene-select`**：基于画像推荐 3-5 个场景卡片，支持自定义输入和手动换一批重新生成。画像完成后自动触发。
- **`practice`**：生成情景练习题，支持 6 种题型。选场景后自动触发。
- **`grade`**：批改用户答案，给出评分 + 解析 + 错误标签。提交答案后自动触发。
- **`explain`**：针对题目/语法/词汇做深入解析。用户追问时触发。
- **`review`**：查询学习记录，生成文字版学习报告。用户查看或一轮结束时触发。
- **`retry`**：基于历史薄弱点生成新题。用户复习时触发。
- **`general-chat`**：兜底 Skill，处理一般聊天。无法匹配其他 Skill 时默认调用。

### 2.3 AI 调度机制

```
用户输入 / 点击学习菜单
  → 系统校验 learningState 与 action 合法性
  → AI Router 接收（用户文本 + 画像 + 当前题目 + 相关历史/记忆摘要 + Skill 列表 + 当前状态）
  → AI 返回 { skillName, params, confidence, rationale }
  → 系统二次校验 schema / allowedStates / 权限 / 锁定策略
  → Skill.handler(ctx) 流式输出 SkillEvent
  → 前端逐步消费渲染
```

**AI 与系统的边界**：AI 负责内容生成和调度建议（推荐场景、生成题目、批改解析、复盘总结），系统负责确定性动作（登录注册、保存进度、提交答案、新建/归档会话、会话锁定、权限判断）。AI 不能绕过系统状态机直接改变主学习流，也不能读取密码、令牌等敏感字段。

**降级策略**：结构化菜单动作优先走确定性路由；自由文本低置信度时展示自然选项确认。`practicing` / `grading` 中不得降级到 `general-chat`，超时或失败时保持原状态并提供重试/恢复。

**工程无感知，可解释**：前端不出现 `SSE`、`command`、`Skill event` 等工程术语；用户只看到"Echo 正在生成""正在准备练习"等自然文案。同时保留简短解释，如“根据你最近介词错误较多，下一题会练 on/in/at”。

### 2.4 学习流状态机

```
onboarding → scene_selecting → practicing → grading → awaiting_next
                                                         ↓
                                      practicing / scene_selecting / reviewing
                                                         ↓
                                             retry(practicing) / archived
```

- **`onboarding`**：收集用户画像
- **`scene_selecting`**：推荐或等待用户选择场景
- **`practicing`**：出题或等待用户答题
- **`grading`**：正在批改（不可切换主线）
- **`awaiting_next`**：批改完成，等待下一步
- **`reviewing`**：展示复盘
- **`archived`**：历史会话，只读

### 2.5 练习与批改

**6 种题型**：

- **单词填空**（`fill` 模式）：训练关键词汇，如 `I would like to ______ a steak.`
- **短语填空**（`fill` 模式）：训练固定搭配，多个空位
- **半句翻译**（`chat` 模式）：给出前半句，用户补全后半句
- **整句翻译**（`chat` 模式）：只给中文，用户写出完整英文
- **对话接龙**（`chat` 模式）：给上一句英文，翻译下一句
- **选择填空**（`select` 模式）：给出选项，点击选择

**出题节奏**：热身（1-2 题简单）→ 核心训练（3-7 题）→ 错题强化 → 正向收尾。每轮 5-10 题。

**批改原则**：接受同义表达、先肯定后纠正、每次聚焦 1-2 个要点。自动标注 12 种错误标签：`spelling` / `word_order` / `tense` / `preposition` / `article` / `subject_verb_agreement` / `auxiliary_verb` / `collocation` / `politeness` / `literal_translation` / `missing_word` / `extra_word`。

**闭环更新**：每次批改后更新错误标签、掌握度、难度分和下次重练建议；重练题优先覆盖未掌握薄弱点，达标后再进入新场景或更高难度。

**难度自适应**：连续 3 题全对提高难度，连续 3 题有错降低难度，用户直说"太难/简单"立即调整。`difficultyScore`（0-1000）对用户不可见。

### 2.6 数据模型

**核心实体**：

- **`users`**：账号（email, password_hash）
- **`user_profiles`**：画像（name, age, grade, level, weakness_tags, recent_topics）
- **`conversations`**：会话（status, learning_state, active_skill, input_mode, lock_policy）
- **`messages`**：消息（type, role, skill_name, content, widget_snapshot, stream_events）
- **`branch_threads`**：辅助追问线程（source_message_id, source_ref）
- **`exercise_attempts`**：练习索引（question_type, prompt, user_answer, status）
- **`grading_results`**：批改索引（score, is_correct, corrections）
- **`error_tag_events`**：错误标签事件（tag, severity, included_in_stats）
- **`mastery_records`**：掌握度记录（tag, mastery_score, next_review_at）
- **`agent_runs`**：Agent 执行记录（run_id, skill_name, status, latency_ms, error_type）

> 复盘、重练、薄弱点统计从结构化表读取，不从自然语言消息中解析。消息只负责呈现，结构化记录负责学习闭环。

### 2.7 流式通信

Skill 通过 `AsyncIterable<SkillEvent>` 输出 8 种事件：

- **`text-chunk`**：文本增量
- **`widget-init`**：初始化 Widget
- **`widget-update`**：更新 Widget 字段
- **`widget-ready`**：Widget 可交互
- **`mode-switch`**：切换输入模式
- **`quick-actions`**：快捷按钮
- **`done`**：完成
- **`error`**：错误

连接链路：`POST /api/chat/send` 创建消息 → `GET /api/chat/stream` SSE 推送 SkillEvent。断线从最后 `seq` 重连。

## 3. 产品特点

### 3.1 会话锁定与防抄袭

- 系统根据学习状态管理会话生命周期：一轮练习结束可自动归档并新建会话，练习进行中主动锁定旧会话
- `practicing` / `grading` 期间，历史会话中的用户答案、参考答案和批改详情默认隐藏，防止练习中翻看旧答案抄袭
- 进入 `awaiting_next` 或 `reviewing` 后，历史详情恢复完整可见，支持完整复盘
- `archived` 会话不可继续答题，只能复盘或作为新学习流的上下文引用
- 锁定是系统规则，不由 AI 决定；AI 只能解释为什么暂时不可查看

### 3.2 辅助追问

- 用户可对当前正在处理的内容或之前任意历史消息、题目、批改结果、语法标签或 Widget 打开右侧支线追问
- 继承源上下文，可连续追问，形成解释链（如"为什么用 on 不用 in？" -> "那 in the morning 为什么是 in？"）
- **不能**改变主学习流状态、生成下一题或替用户提交答案——主线始终不受干扰
- 答题未提交前，辅助追问只提供提示和概念解释，不展示当前题标准答案、完整翻译或等价答案
- 默认不计入薄弱点统计；只有用户明确选择"加入复盘"时才写入结构化记录

### 3.3 Agent 质量与安全

- 所有 Skill 调用记录 `agent_runs`，用于追踪路由命中、状态转移、工具调用、耗时和失败原因
- 关键动作采用 schema 校验、权限校验和状态机校验；高风险或敏感操作必须由用户显式确认
- 用户可反馈“批改不准确 / 题目太难 / 解释看不懂”，反馈进入后续题目生成、批改规则和提示词迭代
- 核心质量指标：Router 命中率、批改一致性、错误标签准确率、重练达标率、用户中断率

### 3.4 性能目标

- **首屏加载**：< 2s
- **流式首字节**：< 1s
- **AI 批改首字**：< 3s
- **新用户注册到第一道题**：< 2 分钟
- **消息丢失率**：0%

## 4. UI 设计

### 4.1 设计方向

暖陶土色 + 纸张感背景。用户觉得在和 AI 教练对话，而非操作学习平台。参考 Claude / Codex 工作流风格。

### 4.2 色彩体系

- **画布底色** `#faf9f5`：奶油白，非纯白
- **AI 消息底色** `#faf9f5`：象牙白 + 左侧陶土色竖线
- **用户消息底色** `#ffffff`：白色 + 暖色边框
- **主强调** `#cc785c`：珊瑚/陶土色（CTA、发送按钮）
- **正确** `#5db872`：绿色
- **错误** `#c64545`：红色
- **正文** `#3d3d3a`：暖深灰
- **标题** `#141413`：近黑
- **辅助文字** `#6c6a64`：灰色
- **分割线** `#e6dfd8`：暖色分割

### 4.3 字体

- **标题**：Copernicus / Tiempos Headline / Cormorant Garamond，28-64px / 字重 400 / 负字距
- **正文**：StyreneB / Inter，14-16px / 字重 400
- **标签**：StyreneB / Inter，12-13px / 字重 500
- **代码**：JetBrains Mono，14px / 字重 400

### 4.4 整体布局

**桌面端（≥ 1180px）**：三栏布局

```
┌──────────────────────────────────────────────────────┐
│ Echora                                        [头像]  │  48px 顶栏
├────────────┬──────────────────────┬───────────────────┤
│ 历史会话    │     主学习流          │   辅助追问         │
│ 240px      │     720px           │   360px           │
│            │                     │                   │
│ · 当前练习  │ 🤖 AI 消息 / Widget  │ 来自：这次批改      │
│ · 校园对话  │ 👤 用户消息          │ 解析 / 追问        │
│ · ...      │ 🤖 批改卡片          │ 子聊天消息流        │
│            │                     │                   │
│            ├──────────────────────┤                   │
│            │ [☰] 输入区 [发送]     │                   │
└────────────┴──────────────────────┴───────────────────┘
```

**移动端（< 768px）**：全宽单栏，历史和辅助追问收进抽屉。

### 4.5 消息气泡

- **AI 消息**：左对齐、max-width 85%、象牙白底色、左侧 3px 陶土色竖线、圆角 8px
- **用户消息**：右对齐、max-width 75%、白色底色、暖色边框、圆角 8px
- **系统消息**：居中、小号字体、浅色

### 4.6 输入区模式

- **`chat` 模式**（默认）：自由文本输入，左侧菜单按钮 + 右侧发送按钮。Enter 发送，Shift+Enter 换行。AI 生成中发送按钮变为停止按钮。
- **`fill` 模式**（填空题触发）：上方显示句子模板，空位用高亮标记，输入框聚焦空位。多个空位时 Tab 切换。
- **`select` 模式**（场景选择触发）：输入区变为选项按钮组，点击即触发。"自定义"点击后切回 `chat` 模式。
- **`menu` 模式**（点击菜单按钮）：浮层菜单，包含"开始练习 / 查看复盘 / 复习薄弱点 / 换场景 / 保存进度"。点击后发送结构化动作。Esc 或再次点击关闭。

### 4.7 Widget 组件

12 种 Widget 嵌入 AI 消息卡片中，统一遵守 `LearningWidget` 协议：

- **`scene-cards`**：场景推荐卡片组（水平排列，emoji + 标题 + 知识点 + 难度），hover 轻微上浮
- **`exercise-card`**：练习题主卡片（题干 + 上下文 + 答题入口）
- **`fill-blank`**：填空输入区域
- **`choice-question`**：选择题按钮组
- **`grading-result`**：批改结果卡片（顶部状态条颜色区分对错、得分大字、用户答案 vs 参考答案对比、中文解析、错误标签 chips、底部快捷按钮）
- **`progress-summary`**：学习进度摘要（题数 + 平均分 + 强弱项 + 建议）
- **`answer-review`**：单题回看卡片
- **`intent-confirm`**：低置信度确认卡片（2-3 个自然选项）
- **`learning-menu`**：输入框左侧学习菜单
- **`account-gate`**：登录/注册/保存进度提示
- **`follow-up-source`**：辅助追问来源提示（"来自：这次批改"）
- **`conversation-lock`**：历史答案锁定提示

**Widget 生命周期**：`loading`（骨架屏）→ `ready`（可交互）→ `disabled` / `submitted` / `expired` / `error`

### 4.8 状态指示

- **AI 生成中**：消息末尾闪烁光标 █，Widget 骨架占位，发送按钮变停止按钮，顶部显示"Echo 正在生成"
- **等待答题**：输入区激活，发送按钮恢复
- **批改中**："AI 正在批改..."，流式呈现结果
- **网络异常**：底部红色提示"发送失败，点击重试"或"正在恢复连接..."
- **会话锁定**："完成当前题后可查看完整答案"

### 4.9 响应式断点

- **≥ 1180px**：三栏（左 240px + 中 720px + 右 360px），Widget 2-3 列
- **768-1179px**：主聊天优先，侧栏可折叠，Widget 2 列
- **< 768px**：全宽单栏，侧栏为抽屉，Widget 1-2 列或水平滑动

移动端软键盘弹起时消息区自动滚到底部，输入区跟随键盘上移。

### 4.10 V1 不做的事

- 不做暗色模式、消息搜索、消息长按菜单、自定义主题
- 不做独立学习页面（Dashboard、练习页、复盘页、错题本页）
- 不做语音/听力——纯文本
- 不做 ECharts 图表——数据以文字摘要 + 简单进度环呈现
- 不做游客模式、用户手输命令、用户自定义变体重练
