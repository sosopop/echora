# 02. 用户画像与个性化场景

## 1. 初始信息收集（Onboarding Skill）

### 1.1 设计原则

不通过表单页面收集信息，而是用对话方式自然获取。AI 逐个询问关键信息，用户以自然语言回答。整个过程 3-5 轮对话。

### 1.2 收集的信息

| 字段 | 说明 | 用途 | 是否必填 |
|------|------|------|----------|
| `name` | 称呼（可填昵称） | 对话中称呼用户 | 否 |
| `age` | 年龄 | 判断认知阶段，影响场景推荐 | 是 |
| `gender` | 性别 | 仅用于称呼、代词或用户主动表达的个性化偏好，不做刻板场景推荐 | 否 |
| `grade` | 年级或学历 | 如"初三""大二""已工作"，判断英语学习阶段 | 是 |
| `englishLevel` | 自评英语水平 | 如"及格线""四级""雅思6分""日常交流没问题" | 是 |
| `levelEvidence` | 水平佐证（可选） | 如"中考英语 108""四级 520"，辅助精确定级 | 否 |
| `learningGoal` | 学习目标（可选） | 如"出国旅游""应对考试""工作沟通" | 否 |

### 1.3 对话收集流程

```text
AI: 嗨！我是你的英语教练 Echo。先简单认识一下你吧～

    你叫什么名字？今年多大了？在读几年级（或者已经工作了）？

用户: 我叫小明，15岁，初三

AI: 小明你好！初三正是英语打基础的关键时候。
    你英语大概是什么水平？比如考试能拿多少分，或者自己感觉怎么样？

用户: 中考模拟英语大概 100 分左右，满分 120。语法比较弱。

AI: 明白了。语法弱是很多同学的共同问题，多练练句型就好了。
    你学英语主要是为了考试，还是自己也想用起来？

用户: 主要是中考吧，但我也想能开口说几句

AI: 了解了！那我帮你梳理一下：

    📋 你的学习档案
    · 姓名：小明
    · 年龄：15 岁 / 初三
    · 水平：约 A2-B1（中考 100/120）
    · 短板：语法
    · 目标：中考 + 日常交流

    根据你的情况，我推荐这几个场景，看看有没有感兴趣的——
    ↓
    进入场景选择（scene-select Skill）
```

### 1.4 定级规则

AI 根据用户提供的水平佐证，映射到内部 CEFR 等级：

| 用户描述 | 推断 CEFR | 初始难度分 |
|----------|-----------|------------|
| 小学 / 刚接触英语 | A1 | 80 |
| 初中 / 及格线附近 | A2 | 200 |
| 高中 / 100-120/150 | A2-B1 | 320 |
| 大学四级 / 雅思 5-5.5 | B1 | 420 |
| 大学六级 / 雅思 6-6.5 | B2 | 580 |
| 专八 / 雅思 7+ / 留学 | C1 | 780 |

如果用户无法描述水平，AI 可以通过 3-5 道简单热身题快速评估，或者在后续练习中隐式校准。热身题仍按正式练习记录写入 `exercise_attempts` 与 `grading_results`，不新增独立测评流程表。

### 1.5 跳过机制

用户可以跳过 onboarding 中的任何非必填问题：
- "不想说" / "随便" / "先不管" → AI 跳过该问题，使用默认值
- "直接开始吧" → AI 跳过所有剩余问题，使用 age=25, level=A2 作为默认画像

## 2. 场景推荐（Scene-Select Skill）

### 2.1 推荐逻辑

AI 基于用户画像生成 3-5 个个性化场景建议：

```text
输入：
  - 年龄：15
  - 年级：初三
  - 水平：A2-B1
  - 短板：语法
  - 目标：中考 + 日常交流

AI 推理：
  - 初三学生 → 校园、日常、考试相关场景
  - 语法弱 → 场景应包含目标语法点的自然重复
  - 中考目标 → 覆盖考纲常见话题（饮食、购物、问路、健康、环保等）
  - 也想开口 → 加入实用口语场景

输出（3-5 个场景）：
  1. 🍔 餐厅点餐 — 练习 would like / 一般疑问句
  2. 🏫 校园对话 — 练习一般现在时 / 情态动词
  3. ✈️ 旅行问路 — 练习 there be / 介词搭配
  4. 📚 图书馆借书 — 练习 can/could / 礼貌请求
```

### 2.2 场景卡片 Widget

场景推荐以卡片形式嵌入对话，用户可点击选择：

```
┌──────────────────────────────────────────┐
│ 🤖 Echo                                  │
│                                          │
│ 根据你的情况，推荐这几个场景：              │
│                                          │
│ ┌──────────────┐ ┌──────────────┐        │
│ │     🍔       │ │     🏫       │        │
│ │  餐厅点餐     │ │  校园对话     │        │
│ │              │ │              │        │
│ │ 练习：        │ │ 练习：        │        │
│ │ would like   │ │ 一般现在时    │        │
│ │ 一般疑问句    │ │ 情态动词      │        │
│ │ B1 难度      │ │ A2 难度       │        │
│ └──────────────┘ └──────────────┘        │
│ ┌──────────────┐ ┌──────────────┐        │
│ │     ✈️       │ │     📚       │        │
│ │  旅行问路     │ │  图书馆借书    │        │
│ │              │ │              │        │
│ │ 练习：        │ │ 练习：        │        │
│ │ there be     │ │ can/could    │        │
│ │ 介词搭配      │ │ 礼貌请求      │        │
│ │ A2 难度      │ │ A2 难度       │        │
│ └──────────────┘ └──────────────┘        │
│                                          │
│ [✨ 随便来一个]  [📝 自定义场景...]       │
└──────────────────────────────────────────┘
```

桌面端：卡片水平排列（2-3 列）
移动端：卡片垂直堆叠或水平滑动
用户对某个推荐场景有疑问时，可从卡片打开右侧辅助追问，追问推荐理由、可训练表达或替代场景；主聊天仍停留在场景选择。

`scene-cards` 必须遵守统一 Widget 协议：点击卡片只发送结构化 action，系统确认当前 `learningState` 允许进入练习后，再触发后续 Skill。

### 2.3 自定义场景

用户可以通过以下方式自定义场景：
- 点击"自定义场景"按钮 → 输入框切换为 chat 模式 → 用户输入自然语言描述
- 直接在对话框中输入 "我想练 XX"

AI 将自定义输入规范化为场景：
```json
{
  "titleZh": "向老师请教数学问题",
  "titleEn": "Asking a teacher for help with a math problem",
  "category": "campus_english",
  "estimatedLevel": "B1",
  "targetExpressions": ["can you help me with...", "I don't understand...", "could you explain..."]
}
```

### 2.4 场景去重

- 系统从 `user_profiles.recentSceneTopics` 和最近 `exercise_attempts` 维护最近使用的场景主题列表（最近 10 个）
- 推荐场景时避开最近主题，AI 只负责生成候选与说明
- 用户说"继续练"时，AI 自动避开最近主题

## 3. 用户画像更新

用户画像不是一成不变的。随着练习数据积累，画像自动更新：

| 字段 | 更新方式 |
|------|----------|
| `englishLevel` / `difficultyScore` | 根据练习成绩自动调整 |
| `weaknessTags` | 根据 `error_tag_events` 聚合 |
| `strengthTags` | 根据连续正确的知识点标记 |
| `scenarioPreferences` | 根据用户选择的场景类别统计 |
| `totalExercises` | 根据 `exercise_attempts` 累计计数 |

用户可以通过对话查询自己的画像（"我最近水平有变化吗？"），AI 用自然语言描述，不展示原始数据。
