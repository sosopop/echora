# 00. V1 MVP 产品范围

## 1. 产品定位

Echora 是一个 AI 英语练习应用。V1 MVP 的形态是：**一个连续学习流工作台 + 一套内部可插拔能力体系**。

所有功能——场景选择、练习、批改、复盘、登录注册、历史回顾——都可以封装为内部 Skill 或系统动作。AI 根据用户当前状态给出内容、解释、推荐和调度建议；系统负责登录注册、保存、新建会话、提交答案、会话锁定、权限判断等确定性动作。用户只看到自然对话、学习菜单、题目输入和小部件，不看到路由、命令或传输协议。

用户感知到的始终是"和一个聪明的英语教练对话"。桌面端可以看到历史对话列表和右侧辅助追问，但这些只是帮助用户管理上下文，不把产品变成传统学习管理系统。

## 2. Skills 架构原则

### 2.1 一切皆 Skill

每个功能模块封装为一个 Skill。Skill 是独立的功能单元，包含：

- **元数据**：名称、描述、触发条件（供 AI 路由决策）
- **输入 schema**：定义接受的参数
- **处理器**：执行逻辑，支持流式输出
- **UI 定义**（可选）：Widget 组件、输入框模式

### 2.2 AI 建议 + 系统确定动作

用户输入和点击学习菜单产生的动作先经过系统校验：当前学习流状态是否合法、是否需要确认、是否需要锁定或解锁历史。AI 根据对话上下文和可用 Skill 列表做意图识别，给出调用建议；系统再决定是否执行对应 Skill 或确定性动作。用户不需要知道有哪些 Skill、命令或路由存在。

### 2.3 Skill 可插拔

Skill 通过注册机制加入系统。新增功能 = 新增 Skill，不改动核心路由逻辑和聊天框架。这类似于 MCP（Model Context Protocol）的工具模型。

### 2.4 Widget 渐进式呈现

复杂交互（如场景选择、填空输入）不是跳转页面，而是以 Widget 形式嵌入对话流。Widget 由 Skill 或系统动作声明，前端根据统一 Widget 协议动态渲染。所有 V1 Widget 必须遵守 `08-ai-workflow-and-widget-contract.md` 中的字段、生命周期、动作输出和持久化规则。

### 2.5 输入模式自动切换

输入框不是一成不变的文本域——根据当前激活的 Skill，自动切换为不同的交互模式（自由文本 / 填空 / 选项 / 学习菜单）。

### 2.6 历史会话与辅助追问

主聊天流保留完整学习路径；左侧历史对话列表用于切换 active / archived 会话。AI 可根据学习状态自动新建会话；练习进行中，旧会话默认只读或暂不可进入，避免用户回看答案抄袭；复盘阶段恢复完整历史查看。用户可以从任意消息、题目、推荐答案、语法标签或 Widget 内容打开右侧辅助追问。辅助追问继承源消息上下文，用于解释、扩展表达、追问语法或生成相似例句，不打断中间主练习流程。

### 2.7 用户可见文案边界

前端用户界面不得出现 `SSE`、`command`、`AI Router confidence`、`Skill event` 等工程术语。技术实现可以继续使用内部流式传输、AI 路由和 Skill 事件，但用户只看到"Echo 正在生成"、"正在准备练习"、"学习进度已保存"、"需要你确认下一步"等自然文案。

### 2.8 工作流契约优先级

学习流状态机、AI/系统边界、Widget 协议、辅助追问和会话锁定规则以 `08-ai-workflow-and-widget-contract.md` 为实现基准。本文档只定义产品范围，不重复展开每个 Widget 的字段契约。

## 3. V1 MVP 功能边界

### 3.1 包含的 Skill

| Skill | 说明 | 触发方式 |
|-------|------|----------|
| `onboarding` | 收集用户基础画像（年龄、性别、姓名、年级、英语水平），生成个性化场景建议 | 首次登录自动触发 |
| `scene-select` | 根据用户画像展示 3-5 个场景卡片供选择，支持自定义输入 | onboarding 完成后自动触发，或用户说"换个场景" |
| `practice` | 生成情景对话练习题，支持多种出题方式 | 场景确定后自动触发，或用户说"开始练习" |
| `grade` | 批改用户答案，给出评分和解析 | 用户提交答案后自动触发 |
| `explain` | 针对某道题、某个语法点或词汇做深入解析 | 用户追问时自动触发 |
| `review` | 查询学习历史，生成文字版学习报告 | 用户说"查看进度"或点击学习菜单 |
| `retry` | 针对历史薄弱点生成新题 | 用户说"复习错题"或点击学习菜单 |
| `general-chat` | 兜底 Skill，处理一般聊天 | 无法匹配其他 Skill 时默认调用 |

### 3.2 包含的 Widget

| Widget | 说明 | 对应 Skill |
|--------|------|------------|
| `scene-cards` | 场景选择卡片（水平滚动，3-5 个） | `scene-select` |
| `exercise-card` | 练习题主卡片，承载题干、参考上下文和答题入口 | `practice` |
| `fill-blank` | 填空题内联编辑器 | `practice`（填空模式） |
| `choice-question` | 选择题或确认题按钮组 | `practice` / `scene-select` |
| `grading-result` | 批改结果卡片（正确/错误、解析、标签） | `grade` |
| `progress-summary` | 学习进度摘要 | `review` |
| `answer-review` | 单题回看卡片，展示题目、用户答案、批改和追问入口 | `review` / `retry` |
| `intent-confirm` | 低置信度或高风险动作确认卡片 | 系统通用 |
| `learning-menu` | 输入框左侧学习菜单，提供开始练习、查看复盘、复习薄弱点、换场景、保存进度等入口 | 系统通用 |
| `account-gate` | 在对话流中完成登录、注册、保存进度等账号动作 | `onboarding` / 系统通用 |
| `follow-up-source` | 右侧辅助追问的来源提示，说明来自哪条消息、题目或批改 | `explain` |
| `conversation-lock` | 历史会话受限提示，说明当前练习完成前哪些内容暂不可看 | 系统通用 |

### 3.3 输入框模式

| 模式 | 表现 | 触发条件 |
|------|------|----------|
| `chat` | 自由文本输入，占位提示鼓励自然语言 | 默认模式 |
| `fill` | 显示句子模板，空位高亮，输入焦点在空位 | `practice` Skill 出填空题时 |
| `select` | 输入区变为选项按钮组，点击选择 | `scene-select` Skill 展示场景时 |
| `menu` | 输入框左侧按钮打开学习菜单，点击后发送结构化动作 | 用户点击菜单按钮 |

### 3.4 不包含（V1 MVP）

- 独立学习页面（Dashboard、练习页、复盘页、错题本页）——全部取消，仍然只保留一个聊天工作台页面
- 多主流程页面并排——不做；桌面端仅允许辅助布局：左侧历史会话、中间主聊天、右侧辅助追问
- ECharts 图表——数据以文字摘要 + 简单进度环呈现
- 100 个候选主题预生成——场景按需即时生成
- 语音/听力——纯文本
- 用户手动设置难度——AI 隐式自适应
- 变体重练——暂不做
- 游客模式——暂不做
- 用户手输命令——暂不做，所有快捷功能通过学习菜单呈现

## 4. 用户旅程

### 新用户首次使用

```text
打开 Echora → 注册/登录
  ↓
AI: "嗨！我是你的英语教练 Echo。开始之前，先简单了解一下你——"
AI: "你叫什么名字？今年多大？读几年级（或已毕业）？"
用户: "我叫小明，15岁，初三"
  ↓
AI: "你之前英语学得怎么样？有没有考过什么试？"
用户: "考过中考模拟，英语大概及格线上下"
  ↓
AI: "了解了。根据你的情况，我推荐这几个场景给你试试——"
  ↓
[对话中出现 3 个场景卡片 Widget]
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │ 🍔 餐厅点餐 │ │ 🏫 校园对话 │ │ ✈️ 旅行问路 │
  └──────────┘ └──────────┘ └──────────┘
  [ 或者输入你想练的场景...         ]
  ↓
用户点击"餐厅点餐"
  ↓
AI: [生成餐厅场景练习，逐题出题]
  ↓
练习 → 批改 → 追问 → 继续...
```

### 老用户返回

```text
登录 → 恢复最近对话
  ↓
AI: "小明，欢迎回来！上次你练了餐厅点餐，介词还需要巩固。要接着练，还是试试新场景？"
  ↓
[3 个推荐场景卡片 + "继续上次"快捷按钮]
```

## 5. 关键度量

- 新用户从注册到开始第一道题 < 2 分钟
- AI 内部能力调度准确率 > 80%
- 用户不需要阅读任何文档即可使用
