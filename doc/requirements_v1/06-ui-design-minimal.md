# 06. 极简 UI 设计

## 1. 设计方向

V1 MVP 只有一个页面：聊天工作台。设计目标是：**用户觉得自己在和一位 AI 英语教练持续学习，通过对话流、小部件和底部交互区完成所有操作，而非操作一个学习平台**。

延续暖陶土色 + 纸张感背景的视觉语言。整体交互参考 Claude Code / Codex 的工作流感：左侧是历史上下文，中间是主学习流，右侧是辅助追问；底部交互区既能输入自然语言，也能通过一个普通菜单触发常用学习动作。

## 3. 色彩与字体

沿用原设计系统的暖色调体系：
- 页面背景：`#f5f4ed`（parchment）
- AI 消息卡：`#faf9f5`（ivory）+ 左侧陶土色边线
- 用户消息：`#ffffff` + 暖色边框
- 主强调：`#c96442`（terracotta）
- 正确绿：`#2f8f5b`
- 错误红：`#b53333`
- 字体：Inter / PingFang SC / Georgia（serif 仅用于场景摘要）

## 4. 消息气泡

### 4.1 AI 文本消息

```
┌──────────────────────────────────┐

│ 你好小明！根据你的情况，           │
│ 我推荐这几个场景——                │
```

- 左对齐，max-width 85%

## 5. 输入区设计

### 5.1 chat 模式（默认）

```
┌──────────────────────────────────────────┐
│ [☰] [ 说说你想练什么，或回答当前题... ] [发送] │
└──────────────────────────────────────────┘
```

- 输入框高度 44-160px（自动扩展）
- 左侧按钮打开学习菜单，不展示命令或斜杠语法
- 发送按钮：陶土色底，AI 生成中变为停止按钮

### 5.2 fill 模式（填空题）

```
┌──────────────────────────────────────────┐
│ I would like to _______ a medium steak.  │  ← 句子模板
│                      ████████             │  ← 空位高亮
│ [ return                        ] [✓]   │  ← 答题输入
└──────────────────────────────────────────┘
```

- 句子模板显示在上方，空位用彩色下划线标记
- 输入框聚焦空位答案
- 有多个空位时 Tab 切换

### 5.3 select 模式（选项选择）

```
┌──────────────────────────────────────────┐
│ 选择一个场景开始练习：                     │
│                                          │
│  [🍔 餐厅点餐]  [🏫 校园对话]  [✈️ 旅行]  │  ← 选项按钮
│  [📝 自定义场景...]                      │
└──────────────────────────────────────────┘
```

- 选项按钮：卡片式，点击即选中并触发
- "自定义"点击后切换为 chat 模式

### 5.4 学习菜单（点击输入框左侧按钮）

```
┌─────────────────┐
│ 开始练习        │
│ 查看复盘        │
│ 复习薄弱点      │
│ 换个场景        │
│ 保存学习进度    │
└─────────────────┘
```

- 浮层在输入框上方
- 点击菜单项立即发送结构化动作
- 看起来像普通菜单，不让用户知道底层命令、路由或 Skill
- Esc 或再次点击菜单按钮关闭

## 6. Widget 组件设计

Widget 是 Echora 的功能承载层。AI 在合适时机选择并展示小部件，用户不需要离开聊天流即可完成网站功能。V1 至少支持场景卡片、练习题、批改结果、复盘摘要、学习菜单、账号保存/登录注册提示、意图确认和辅助追问入口。

所有 Widget 都遵守 `08-ai-workflow-and-widget-contract.md` 的统一协议。UI 层不直接信任 Widget 的按钮动作；点击后只发送结构化 action，由系统校验状态、权限和会话锁定后再执行。

### 6.0 Widget 状态与清单

| Widget | UI 表达 | 主线影响 |
|--------|---------|----------|
| `scene-cards` | 场景卡片组 | 选择后进入场景/练习 |
| `exercise-card` | 题干、上下文、答题要求 | 创建正式练习 |
| `fill-blank` | 空位高亮输入 | 提交后进入批改 |
| `choice-question` | 选项按钮组 | 可用于答题或确认 |
| `grading-result` | 得分、参考答案、错因、标签 | 批改完成后进入等待下一步 |
| `progress-summary` | 题数、平均分、强弱项、建议 | 复盘展示 |
| `answer-review` | 单题回看与追问入口 | 复盘/重练入口 |
| `intent-confirm` | 二到三个自然选项 | 确认后才改变主线 |
| `learning-menu` | 输入框左侧普通菜单 | 发送结构化动作 |
| `account-gate` | 保存、登录、注册提示 | 系统账号动作 |
| `follow-up-source` | 右侧"来自：这次批改"来源卡 | 不改变主线 |
| `conversation-lock` | 历史受限提示 | 防抄袭展示 |

Widget 生命周期视觉规则：

- `loading`：骨架屏或轻量等待态，文案为"正在准备..."
- `ready`：可操作状态，按钮可点击
- `disabled`：当前学习流不允许操作，展示原因
- `submitted`：答案或选择已提交，保留结果但禁用重复提交
- `expired`：旧 Widget 不再可操作，可查看但需重新生成
- `error`：展示友好错误和重试按钮

### 6.1 场景卡片 (SceneCardsWidget)

- 水平排列，桌面端最多 3 列，移动端滑动
- 每张卡片：icon + 中文标题 + 知识点标签 + 难度标签
- hover 时轻微上浮 + 边框高亮
- 点击选中 → 触发场景选择 action

### 6.2 批改结果 (GradingResultWidget)

- 顶部状态条：绿色（正确）/ 橙色（部分正确）/ 红色（错误）
- 得分大字展示
- 用户答案 vs 参考答案 对比展示
- 错误位置高亮标注
- 中文解析文字
- 错误标签 chips
- 底部快捷按钮

### 6.3 进度摘要 (ProgressSummaryWidget)

- 练习统计：题数、平均分
- 强项列表（绿色标记）
- 弱项列表（橙色标记）
- 建议文字
- 底部快捷按钮

## 7. 状态指示

### 7.1 AI 生成中

- AI 消息末尾显示闪烁光标 █
- Widget 初始化中显示骨架占位
- 发送按钮变为停止按钮（方形、红色边框）
- 输入框禁用
- 顶部状态显示"Echo 正在生成"或"正在准备练习"
- 可展示"AI 处理过程"面板：理解你的目标 → 选择合适练习 → 生成互动卡片 → 准备下一步

### 7.2 等待用户答题

- 练习 Widget 中，输入区激活
- 发送按钮恢复正常

### 7.3 批改中

- 用户答案已显示，下方显示"AI 正在批改..."
- 3s 后流式呈现批改结果

### 7.4 网络异常

- 输入框底部红色提示条："发送失败，点击重试"
- 连接中断提示："正在恢复连接..."

### 7.5 会话锁定

练习处于 `practicing` 或 `grading` 时，左侧历史会话仍可见，但旧会话中的答案、参考答案和批改默认隐藏或只读。UI 用 `conversation-lock` Widget 或历史列表状态提示解释：

- "完成当前题后可查看完整答案"
- "这个会话可复盘，当前练习不受影响"
- "已隐藏参考答案，避免影响正在进行的练习"

复盘阶段恢复完整历史查看。

### 7.6 用户可见文案禁用词

用户界面不得出现 `SSE`、`command`、`AI Router confidence`、`Skill event stream`、`source: msg_042` 等工程术语。对应展示文案应改为"实时生成"、"学习菜单"、"需要确认下一步"、"AI 处理过程"、"来自：这次批改"。

## 8. 空状态

**首次使用（无任何历史消息）**：
- 聊天区顶部显示 AI 首条消息（由 `onboarding` Skill 自动触发）
- 用户看不到任何 UI 控件，只有对话引导

**会话新建后（有画像，无当前对话）**：
- AI 打招呼 + 基于画像直接推荐场景

## 9. 响应式

| 断点 | 消息区宽度 | Widget 列数 | 输入区 |
|------|-----------|-------------|--------|
| ≥ 768px | 720px 居中 | 2-3 列 | 720px 居中 |
| ≥ 1180px | 中间 720px，左 240px，右 320-360px | 2-3 列 | 中间 720px，辅助追问独立输入 |
| 768-1179px | 主聊天优先，历史/辅助追问可折叠 | 2 列 | 主输入 720px 内 |
| < 768px | 100% - 24px | 1-2 列或水平滑动 | 全宽，历史/辅助追问为抽屉 |

移动端软键盘弹起时：消息区自动滚到底部，输入区跟随键盘上移。

## 10. 不做的事（V1 MVP）

- 不做暗色模式
- 不做消息搜索
- 不做消息长按菜单
- 不做传统消息引用回复；只支持右侧辅助追问
- 不做多个主会话并排；左侧只用于历史切换
- 不做动画回放
- 不做自定义主题
- 不让用户直接输入或学习命令
