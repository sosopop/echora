# 03. 练习与学习闭环

## 1. 练习 Skill（Practice）

### 1.1 触发方式

- 用户选择场景后自动触发
- 用户说"开始练习"、"继续练"、"再来一轮"
- 用户点击学习菜单里的"开始练习"
- AI 感知到用户想练习（如"想练练手"）

### 1.2 练习流程

```text
Practice Skill 激活
  ↓
系统将主学习流置为 practicing，并锁定旧会话答案/批改的可见范围
  ↓
AI 生成场景介绍（流式，2-3 句）
  ↓
AI 生成第 1 道题（根据水平选择题型）
  ↓
输入框切换为对应模式（fill / chat）
  ↓
用户提交答案
  ↓
系统写入 exercise_attempts，并将主学习流置为 grading
  ↓
Grade Skill 批改（流式）
  ↓
系统写入 grading_results 与 error_tag_events，并将主学习流置为 awaiting_next
  ↓
用户可从题目/批改结果打开右侧辅助追问
  ↓
AI 建议：继续出题 / 追加同类题 / 切换题型 / 询问是否继续；系统校验并执行主线状态流转
  ↓
循环...直到用户主动结束或一轮完成（5-10 题）
  ↓
Review Skill 生成本轮小结
```

### 1.3 题型与输入模式

AI 根据用户水平、题目知识点和练习目标动态选择题型。不同题型对应不同的输入模式：

| 题型 | 输入模式 | 示例 | 适用场景 |
|------|----------|------|----------|
| 单词填空 | `fill`（单个空） | `I would like to ______ a steak.` | 训练关键词汇，低水平用户 |
| 短语填空 | `fill`（多个空） | `I ______ ______ ______ a medium steak.` | 训练固定搭配 |
| 半句翻译 | `chat` | 给出前半句 `I'd like...`，用户补全后半句 | 训练句子结构 |
| 整句翻译 | `chat` | 只给中文："我想要一份五分熟的牛排。" | 训练完整表达 |
| 对话接龙 | `chat` | 给上一句英文，翻译下一句 | 训练上下文反应 |
| 选择填空 | `select` | 给出 3 个选项，用户点击选择 | 低水平用户或快速复习 |

### 1.4 AI 出题节奏

AI 不是机械覆盖所有句子，而是根据学生状态动态调整：

1. **热身**：前 1-2 题偏简单，建立信心
2. **核心训练**：3-7 题围绕目标知识点
3. **错题强化**：如果用户答错，追加 1-2 道同类型题
4. **收尾**：最后 1 题轻松，正向结束

### 1.5 用户控制

练习中用户可随时：

| 操作 | 方式 |
|------|------|
| **跳过** | 点快捷按钮"跳过"或说"下一题" |
| **要提示** | 点快捷按钮"给提示"或说"提示一下" |
| **追问** | 点题目/批改卡片的"为什么"或"解析"按钮，在右侧辅助追问中继续 |
| **切换场景** | 说"换个场景" → 触发 `scene-select` Skill |
| **结束练习** | 说"不练了" → 触发 `review` Skill 小结 |
| **调整难度** | 说"太难了"或"太简单了" → AI 立即调整 |

## 2. 批改 Skill（Grade）

### 2.1 批改流程

```text
用户提交答案 → 系统创建 exercise_attempts → Grade Skill 激活
  ↓
输入：题目上下文 + 用户答案 + 参考答案 + 可接受答案列表
  ↓
AI 流式生成批改结果：
  1. 先输出判断（正确/部分正确/错误）→ 前端立即展示颜色标识
  2. 再输出得分
  3. 再输出解析文字（中文）
  4. 最后附加错误标签
  ↓
批改卡片 Widget 渲染在对话中
快捷按钮："为什么？"（打开右侧辅助追问）"再来一道"
  ↓
系统持久化 grading_results 和 error_tag_events，供复盘、重练、防抄袭和薄弱点统计使用
```

### 2.2 批改卡片 Widget

```
┌─────────────────────────────────┐
│ ✅ 不错 · 85 分                   │
│                                 │
│ 你的答案：                       │
│ I want a medium steak            │
│ with fries and salad.            │
│                                 │
│ 参考答案：                       │
│ I'd like a medium steak          │
│ with fries and a salad.          │
│                                 │
│ 📝 分析                          │
│ 意思完全正确！不过有两个小地方      │
│ 可以让表达更地道：                │
│                                 │
│ ① "I want" → "I'd like"         │
│   点餐时说 I'd like 更有礼貌      │
│                                 │
│ ② "salad" → "a salad"           │
│   可数名词别忘加冠词 a            │
│                                 │
│ 🏷 介词 · 冠词                    │
│                                 │
│ [💡 为什么 want 不够礼貌？]       │  → 打开右侧辅助追问
│ [🔄 再来一道类似的]               │
└─────────────────────────────────┘
```

### 2.3 评分原则

- 不只看字符串匹配，接受同义表达
- 先肯定正确的部分，再指出可改进的
- 每次批改聚焦 1-2 个要点，不过载
- 用语温和，面向学生而非评审

### 2.4 错误标签

AI 自动批注错误标签，用于薄弱点统计：

`spelling` `word_order` `tense` `preposition` `article` `subject_verb_agreement` `auxiliary_verb` `collocation` `politeness` `literal_translation` `missing_word` `extra_word`

## 3. 辅助追问 Skill（Explain）

### 3.1 触发

- 用户点击批改卡片下的快捷按钮（如"为什么？"），右侧打开辅助追问
- 用户点击练习中的某个单词、句子、语法标签或推荐答案
- 用户在右侧辅助追问中继续输入问题（如"介词 in/on/at 到底怎么分"）

### 3.2 行为

`explain` Skill 获取源上下文（原题、用户答案、批改结果、Widget 片段或场景卡片）+ 用户追问内容，生成针对性解释。解释显示在右侧辅助追问，不打断中间主练习流；必要时可把辅助追问解释插回主聊天，作为一条摘要消息保存。

辅助追问只解释来源上下文，不直接生成下一道主线练习题、不提交答案、不改变 `learningState`。如果用户在辅助追问中要求"换个场景"、"再来一题"或"开始复盘"，系统必须在主学习流中展示确认后再执行。

### 3.3 追问链

用户可以在右侧辅助追问中连续追问，形成解释链：

```text
右侧追问来源：批改卡片 msg_042 / errorTag=preposition
用户: 为什么这里用 on 不用 in？
  ↓
AI: [解释 on 和 in 在时间表达中的区别，配 2 个例句]
  ↓
用户: 那 "in the morning" 为什么是 in？
  ↓
AI: [解释固定搭配 in the morning/afternoon/evening，但 on Monday morning]
  ↓
...
```

每一次辅助追问都是独立的消息，链接到原始题目。默认不计入薄弱点统计；只有用户明确选择"加入复盘"或系统识别为正式学习事件时，才写入结构化统计。

## 4. 复盘 Skill（Review）

### 4.1 触发

- 用户说"看看我的进度"、"最近练得怎么样"
- 用户点击学习菜单里的"查看复盘"
- 完成一轮练习后 AI 主动触发（小结模式）

### 4.2 小结模式（练习后自动触发）

```text
AI: 这轮练习完成！来看看你的表现：

    ✅ 完成 8 道题
    📊 平均分 82
    💪 做得好：词汇选择准确，语序正确
    🎯 还需加强：冠词（a/an/the）漏了 3 次，点餐敬语可以更地道

    建议接下来练一下"酒店入住"，那里也有很多冠词使用场景。

    [继续练习] [换个场景] [查看详细报告]
```

### 4.3 完整复盘模式（用户主动查询）

用户点击"查看复盘"或说"给我看看最近一周的情况"时，AI 生成详细报告：

- 练习天数、总题数、平均分
- 进步明显的领域（对比之前）
- 持续薄弱的领域（附具体例子）
- 学习建议

所有以自然语言呈现，不使用图表。

复盘数据从 `exercise_attempts`、`grading_results`、`error_tag_events` 读取，不从自然语言消息或 Widget 展示文本临时解析。

## 5. 重练 Skill（Retry）

### 5.1 触发

- 用户说"把错题再练一下"、"复习一下之前的"
- 用户点击学习菜单里的"复习薄弱点"

### 5.2 行为

1. 系统从 `error_tag_events` 查询用户最近薄弱标签（按错误频率、最近性和严重度排序）
2. 选择 top 1-2 薄弱标签
3. 生成 3-5 道新题（不是原题重做，是新场景 + 相同知识点）
4. 以练习模式呈现

## 6. 难度自适应

### 6.1 隐式调整

难度调整对用户透明，不展示 difficultyScore：

- 连续 3 题全对 → 稍微提高难度（更长的句子、更复杂的语法点）
- 连续 3 题都有错 → 降低难度（更多提示、更简单的题型）
- 用户直接说"太难了"/"太简单了" → 立即调整

### 6.2 内部状态

后端维护 difficultyScore（0-1000）和 CEFR 等级，用于 AI 生成内容时的隐形约束。
